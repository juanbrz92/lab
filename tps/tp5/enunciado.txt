Fecha de entrega: 18/10/16


Enunciado
~~~~~~~~~

El objetivo del práctico es desarrollar un servidor web multi-hilos que pueda atender 
múltiples conexiones simultáneas.


Requisitos
~~~~~~~~~~

* El tp que no tenga el Makefile correspondiente se considerará desaprobado.

* El tp que no tenga como mínimo 5 módulos se considerará desaprobado.

* Que sea configurable vía un archivo de configuración. En el mismo deberán ser parametrizables 
al menos: el puerto por el que atenderá el servidor y la ruta del sistema de archivos que será 
la raíz de la dirección web.

* El nombre del archivo de configuración deberá poder cambiarse como el argumento "-f" del 
archivo ejecutable en cuestión.

* Apache benchmark: ab -c 5 -n 10000

* Debe poder manejar como mínimo los siguientes tipos de archivos: jpg, html, text, pdf, mp3, mp4,
gif, json.

* Debe devolver como mínimo, cinco tipos de resultados al cliente: 200, 400, 403, 404 y 500 (Ver 
especificación de HTTP 1.0).

* El servidor debe manejar como mínimo los métodos GET y POST.


Todo lo que no está especificado en este enunciado queda a consideración de los alumnos.


Uso
~~~

-----------------------------
$ ./tp3 -f <archivo_config>
-----------------------------


Archivo de configuración
~~~~~~~~~~~~~~~~~~~~~~~~

-----------------------
$ cat archivo_config

port=<nro_puerto>
droot=<document_root>
-----------------------


Tips
~~~~

* mime types en /etc/mime.types

* RFC 1945 - Hypertext Transfer Protocol -- HTTP/1.0

* Ver concepto de URI, URL, Mime Yype, cabecera HTTP (request y response), HTTP Status Code, HTTP Body.
